Getting Started

Tutorial

From zero to running autonomous agents in 6 steps.

1

Install claw-forge

We use uv for fast, isolated installation. uv tool install puts the CLI on your PATH without polluting your system Python.

pip install uv
uv tool install claw-forge
claw-forge --version
claw-forge 0.2.0b1

If you prefer pipx: pipx install claw-forge

2

Configure providers

Create claw-forge.yaml in your project root. The minimal setup uses your existing claude login session (zero API key required) plus Groq as a free backup.

# claw-forge.yaml project: name: my-app path: . providers: # Zero config — uses your claude login session - name: claude-oauth type: oauth enabled: true priority: 10 # Direct key — higher RPM, lower priority as backup - name: anthropic-direct type: anthropic api_key_env: ANTHROPIC_API_KEY enabled: true priority: 5 # Groq — free tier, great for lightweight tasks - name: groq-backup type: openai_compat api_key_env: GROQ_API_KEY base_url: https://api.groq.com/openai/v1 model: llama-3.3-70b-versatile enabled: true priority: 2 orchestrator: max_concurrent: 3 # parallel agents retry_attempts: 3

The pool manager automatically routes to the healthiest available provider. If claude-oauth hits rate limits, it falls back to anthropic-direct, then groq-backup.

3

Create your project

Write app_spec.txt describing what you want to build. Then initialize the project — the initializer agent reads your spec and creates atomic, dependency-ordered feature tasks.

claw-forge init my-app --spec app_spec.txt
✅ Project analyzed successfully project_name: my-app tech_stack: python/fastapi features_detected: 7 session_manifest: session_manifest.json

Or use the Claude slash command for interactive setup: type /create-spec in any Claude session in your project directory.

4

Run with live output

Start the agent harness. The dispatcher executes features in dependency-ordered waves — independent features run in parallel, bounded by --concurrency.

claw-forge run my-app --concurrency 3
# Watch the Kanban at http://localhost:8888
claw-forge v0.2.0b1 Project: my-app Providers: 3 (claude-oauth 🟢 anthropic-direct 🟢 groq-backup 🟢) Wave 1/3 — 3 tasks (parallel) [claude-oauth] 🤖 coding: Create User SQLAlchemy model ✅ 12s $0.02 [anthropic-direct] 🤖 coding: Implement JWT utility ✅ 18s $0.03 [groq-backup] 🔬 testing: Test infrastructure setup ✅ 8s $0.00 Wave 2/3 — 2 tasks [claude-oauth] 🤖 coding: Add POST /auth/login endpoint ✅ 24s $0.04 [anthropic-direct] 👁 reviewing: Review wave 1 output ✅ 10s $0.01 Wave 3/3 — 2 tasks ...

The Kanban UI at http://localhost:8888 shows real-time status with WebSocket updates, provider health dots, and a running cost tracker.

5

Check provider pool status

See which providers are healthy, their current RPM, and how much each has cost today.

claw-forge pool-status

Use the /pool-status slash command inside Claude for a richer analysis with recommendations.

6

YOLO mode for maximum speed 🚀

When you need to ship fast and you trust the agents, YOLO mode disables human-input prompts, maxes concurrency to your CPU count, and enables aggressive retry (5 attempts).

claw-forge run my-app --yolo
⚠️ YOLO MODE: Human approval skipped, max concurrency (12), aggressive retry claw-forge v0.2.0b1 Project: my-app Providers: 3 Wave 1/3 — 12 tasks (all parallel — CPU count) ...

YOLO mode is perfect for: first-pass feature generation, rebuilding from a clean spec, or when you've already reviewed the agent prompts and trust them.

Pause and resume at any time:

claw-forge pause my-session-id   # drains in-flight agents
claw-forge resume my-session-id  # continues from where it stopped
